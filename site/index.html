01 / The problem changed

The agent is no longer the bottleneck. Human oversight is.

Chat-only execution

The work grows. The record disappears.

  • Intent lives inside a scrolling conversation.
  • Scope changes while the agent is already moving.
  • “Done” is reported, not demonstrated.

A-Team execution

The work grows. Your control does not shrink.

  • Intent becomes a repository-native contract.
  • Every agent gets a bounded claim and Git isolation.
  • Acceptance conditions map to concrete evidence.

02 / From plan to evidence

Keep AI-sized output inside human-sized work packages.

Each step narrows ambiguity and leaves a durable record for the next human decision.

  1. 01

    Define

    Turn intent into an observable outcome, scope and verification contract.

  2. 02

    Package

    Order related tickets by dependency, risk and safe parallelism.

  3. 03

    Claim

    Give one agent one branch and one isolated worktree.

  4. 04

    Review

    Map every acceptance condition to reproducible evidence.

  5. 05

    Close

    Accept only integrated work, then release claims and cleanup safely.

03 / Different layer, different job

Not another chat, tracker, or agent runtime.

A-Team does not replace the tools around it. It supplies the missing control record between human intent and agent execution.

# T-001 — First controlled change

## Outcome
The requested behavior is observable.

## Scope
Implement one bounded product change.

## Non-goals
No adjacent redesign.

## Acceptance
- The observable behavior passes.

## Verification
- Run the focused integration test.

## Constraints
Preserve existing lifecycle semantics.

## Open decisions
None.

## Execution notes
Ready for human review.
